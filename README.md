**Project**: EEG Emotion Recognition with GCNs on DREAMER

- **Overview**: This repository contains code and notebooks used to preprocess EEG trials from the DREAMER dataset, extract Differential Entropy (DE) features, construct channel graphs (Pearson correlation-based), and demonstrate Graph Convolutional Network (GCN) experiments. The work is aimed at emotion recognition (binary valence/arousal classification) using node-level DE features and a shared channel graph.

**Repository Structure**
- `DREAMER.mat`: DREAMER dataset file (MATLAB format) used by the demos and preprocessing.
- `preprocessing.py`: DE feature extraction, baseline correction, normalization, and the main `extract_features_and_labels(...)` function that converts trials into (14,5) DE feature samples.
- `graph_construction.py`: Builds a shared weighted channel graph using Pearson correlation computed on the training set and converts it to PyTorch Geometric (`edge_index`, `edge_weight`) format; produces `Data` objects for training/testing.
- `GCN_demo..py`: Lightweight demo script showing how to load `DREAMER.mat` and extract trial EEG and label arrays (starter script for exploring the dataset). Note: it's a simple example — consider running `print(mat_data.keys())` to inspect structure.
- `GCN_demo.ipynb`, `preprocessing_EDF.ipynb`: Notebooks with interactive experiments and preprocessing explorations.
- `preprocessed_data.pkl`: Example preprocessed output (if present) for quick development.
- `results_gcnn_dreamer_recon.png`: Example result/figure generated by experiments.
- `gcn_venv/`: A project virtual environment created locally (Python 3.11.5). This folder contains a ready Python environment for reproducibility on the same OS and Python version.
- `Models_for_EEG_on_the_Datasets/`: PDFs and papers related to EEG emotion recognition and graph-based models.

**Key Concepts & Pipeline**
- **Feature extraction**: `preprocessing.py` implements DE (Differential Entropy) extraction per 1s segment (128 samples at 128 Hz) across five bands: Delta, Theta, Alpha, Beta, Gamma. Each sample becomes a `(14, 5)` feature matrix (14 channels, 5 band DE features).
- **Normalization**: Z-score normalization applied per-sample in `z_score_normalize(...)`.
- **Baseline correction**: `baseline_correct(...)` subtracts the baseline mean from stimuli signals before feature extraction.
- **Graph construction**: `graph_construction.py` builds a channel adjacency using Pearson correlation on the training set only, converts to absolute weights, adds self-loops, and shares the weighted graph across samples when building `torch_geometric.data.Data` objects.

**Quick Start**
- 1. Activate the provided virtual environment (Windows `cmd.exe`):

  ```cmd
  C:\path\to\repo> gcn_venv\Scripts\activate.bat
  ```

- 2. (Optional) Install or update dependencies inside the venv (see suggested command above).
- 3. Inspect the dataset and labels interactively (recommended):

  ```py
  # run inside a Python shell or notebook
  import scipy.io as sio
  mat = sio.loadmat('DREAMER.mat')
  print(mat.keys())
  ```

- 4. Run preprocessing to extract DE features and labels. The repository includes functions rather than a CLI, so either import `preprocessing.extract_features_and_labels` from a script or notebook, or adapt the demo notebooks.
- 5. Use `graph_construction.py` as a reference to compute the Pearson correlation channel graph from training data, create `torch_geometric.data.Data` objects, and feed them to your GCN training loop.

**Example Workflow (not a single script)**
- Use the following high-level steps to reproduce experiments:
  - Load `DREAMER.mat` and extract per-trial EEG arrays and scores (see `GCN_demo..py`).
  - For each trial, apply baseline correction, truncate/pad last 60s, split into non-overlapping 1s segments and compute DE features using `preprocessing.py`.
  - Split the dataset into train/test (stratified) and compute the Pearson correlation matrix using only training features (see `graph_construction.py`).
  - Convert the dense adjacency to `edge_index` and `edge_weight` and build `torch_geometric.data.Data` objects.
  - Define and train a GCN (not included here) or adapt existing GCN code; evaluate on test `Data` objects.

**Notes & Caveats**
- The demo scripts assume a particular structure inside `DREAMER.mat`. Inspect the loaded `mat` object and adjust indexing accordingly.
- `torch-geometric` installation may require matching PyTorch and CUDA versions; follow the official install instructions at https://pytorch.org and https://pytorch-geometric.readthedocs.io.
- The project currently does not include a ready-to-run training script for a GCN model — the repository contains preprocessing and graph construction utilities and example notebooks to develop experiments.
